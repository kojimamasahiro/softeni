# 対戦成績カード：生成AI用プロンプトとデータ形式

次の対戦が決まったタイミングで投稿する「対戦カード画像」を生成AIで作るためのテンプレート。

## 運用フロー

1. `h2h.py` で対戦データJSONを生成（または手で書く）
2. 下のプロンプトにJSONを埋め込み、画像生成AIに渡す
3. **文字を必ず校正する**（選手名・数字の誤りは画像生成で頻発する）
4. 投稿

```bash
python tools/sns-images/h2h.py --pair-a "水木・松田" --pair-b "丸山・内田" \
    --event "全日本選手権2026" --round "準決勝" > matchup.json
```

## 入力データ形式（JSON）

```json
{
  "event": "全日本選手権2026",
  "round": "準決勝",
  "pairA": {
    "players": ["水木 洸", "松田 拳弥"],
    "team": "東北",
    "prefecture": "宮城県"
  },
  "pairB": {
    "players": ["丸山 海斗", "内田 理久"],
    "team": "NTT西日本",
    "prefecture": "広島県"
  },
  "headToHead": {
    "aWins": 2,
    "bWins": 1,
    "matches": [
      {
        "year": 2024,
        "tournament": "全日本選手権",
        "round": "準々決勝",
        "score": "4-2",
        "winner": "A",
        "retired": false
      }
    ]
  }
}
```

フィールドの意味:

- `event` / `round`: 今回の対戦（カードの見出しになる）
- `pairA` / `pairB`: 対戦するペア。シングルスなら `players` を1人に
- `headToHead.matches`: 過去の対戦履歴（古い順）。`winner` は "A" か "B"
- `score` は常に A側-B側 の順

## プロンプトA：1枚完成型（テキスト描画に強いモデル向け）

Gemini（nano banana）やGPTの画像生成など、文字描画に強いモデル用。それでも日本語の人名は崩れることがあるので、生成後の確認は必須。

```
ソフトテニスの試合告知用SNS画像を1枚、縦長4:5（1200×1500px相当）で生成してください。

【デザイン仕様】
- 背景: 濃いネイビー（#1B2A4A）。ソフトテニスコートを俯瞰した抽象的な線画を薄く敷く
- アクセントカラー: 黄色（#F8B500）。ソフトテニスボールのモチーフとして使用
- フォント: 太いゴシック体、白文字
- 雰囲気: スポーツ中継の対戦カード告知。かっこよく、情報が一目で伝わるように
- 写真や人物の顔は入れない（イラスト・タイポグラフィ中心）

【レイアウト】
1. 最上部: 大会名と回戦（中サイズ）
2. 中央: 左右にペア名を大きく配置し、中央に黄色で「VS」
   - 各ペアの下に所属（チーム名・都道府県）を小さく
3. 中央下: 「過去の対戦成績」として「{aWins} − {bWins}」を大きく表示
4. 下部: 過去の対戦履歴を小さいリストで（年・大会・スコア）
5. 最下部右: 「softeni-pick.com」のクレジット

【使用するテキスト（この文字列のみを正確に使用し、変更・追加・省略しないこと）】
- 大会名: {event}
- 回戦: {round}
- 左ペア: {pairA.players を「・」区切り}
- 左所属: {pairA.team}（{pairA.prefecture}）
- 右ペア: {pairB.players を「・」区切り}
- 右所属: {pairB.team}（{pairB.prefecture}）
- 対戦成績: {aWins} − {bWins}
- 履歴: {matches を「2024 全日本選手権 準々決勝 4-2 ○水木・松田」の形式で1行ずつ}
```

`{...}` をJSONの値で置き換えて使う。AIにJSONごと渡して「上記JSONの値でプレースホルダを埋めて」と指示してもよい。

## プロンプトB：背景のみ生成（文字は後乗せ・確実）

日本語テキストの崩れを完全に避けたい場合。背景だけAIに作らせ、文字はCanva等で乗せる。

```
ソフトテニスの試合告知用SNS画像の「背景」を縦長4:5で生成してください。
文字は一切入れないでください。

- 濃いネイビー（#1B2A4A）基調、黄色（#F8B500）のアクセント
- 中央に対角線状の分割（左右対決の構図）、中央に黄色の円形スペース（後からVSの文字を置く）
- 上部・下部にテキストを置くための余白を確保
- ソフトテニスコートのライン、ラケット、ボールを抽象的な線画モチーフとして配置
- 質感はフラットデザイン寄り、スポーツブランドのポスター風
```

## 補足

- 人名の漢字（髙・濱・﨑などの異体字）は特に崩れやすい。プロンプトAを使う場合も、異体字を含む名前のときはプロンプトBに切り替えるのが安全
- 同一の構図を保ちたい場合は、最初にうまくいった生成画像を参照画像として渡し「テキストだけ差し替え」と指示すると安定する
- X投稿時は1枚目にこのカード、2枚目に勝ち残りドロー画像を付けると回遊が生まれる
