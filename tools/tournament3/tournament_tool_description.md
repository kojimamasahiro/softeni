
# トーナメント管理ツールの説明

このドキュメントは、提供された HTML + JavaScript コードで構成された  
**「トーナメント試合管理ツール」**の機能説明をまとめたものです。

---

## 1. ツールの目的

本ツールは、  
- シングル・ダブルス・団体などのトーナメント形式の試合を  
  **ブラケット形式で管理**し、  
- スコア入力、勝者の自動伝播、BYE処理、パッキン処理、JSON出力  
を行うための管理 UI を提供します。

ブラウザ上で動作し、JSON形式で試合結果をエクスポートできます。

---

## 2. 画面構成（HTML）

### ● ラウンド番号の設定（現在は無効）
```
<input id="roundInput"> / <button id="setRoundBtn">
```
UIとして残っているが、複数ラウンド同時編集モードでは **無効化されている**。

### ● 完了試合数（自動スクロール用）
```
<input id="totalScoredMatchesInput">
```
スコア入力時の自動スクロール調整に利用。

### ● ラウンド名（決勝/準決勝）表示切替
```
<input type="checkbox" id="roundNameToggle">
```

### ● 出力ボタン
```
<button onclick="onClickGenerate()">出力</button>
```
（※ onClickGenerate は元スクリプト依存）

### ● ブラケット表示領域
```
<div id="matches"></div>
```

### ● JSON 出力領域
```
<textarea id="jsonOutput"></textarea>
```

---

## 3. JavaScript の主な機能

### 3-1. ブラケットの生成（buildBracket）
初期プレイヤーを受け取り、

1. 1回戦の全試合を作成  
2. ラウンド数が 1 になるまで空の試合枠を追加  
3. 最終的に「決勝までの全ラウンド」を配列化

```
bracket = [
  { matches: [...] }, // 1回戦
  { matches: [...] }, // 2回戦
  ...
  { matches: [...] }, // 決勝
]
```

---

## 4. パッキン（シード調整）機能

### ● 単体パッキン
1つの試合だけ
```
match.packinMarked = true / false
```
にする。

### ● 一括パッキン
「○番まで」と入力すると、
1回戦の複数の試合に対してまとめてパッキンを設定。

### ● パッキン変更時の挙動
- ブラケットは**再構築**され、  
- プレイヤーの並び替えが行われ、  
- 過去の試合結果は  
  **“p1ID と p2ID の組み合わせ” をキーに復元**される。

---

## 5. スコア入力と勝者の自動伝播

### ● スコア入力ボタン
各選手につき 0〜5 のボタンを用意し、クリックでスコア確定。

### ● 勝者決定
スコアを比較して winner を設定。

### ● propagateWinner
勝者が次のラウンドの対応する枠へ自動的に反映される。

---

## 6. BYE（不戦勝）処理

ID が `"bye"` の選手は BYE と見なす。

- BYE vs 選手 → 選手が自動勝利  
- BYE vs BYE → 編集不可の薄いカード表示

---

## 7. JSON 出力機能

ブラケット全体を整理して JSON 化し、  
`#jsonOutput` に出力する。

外部保存・API連携に利用可能。

---

## 8. 自動スクロールの仕組み

スコア入力後、画面の流れに沿って自動的にスクロールする。

スクロール量は  
- 通常：190px  
- パッキン時：118px  
- フォールバック：190px  
で制御される。

---

## 9. 全体構造まとめ

- フロントエンドだけで動作  
- プレイヤー配列を受け取り自動的にブラケット生成  
- ラウンドを跨いで試合を一括編集可能  
- パッキン（順番調整）による再構築あり  
- JSON による外部連携が可能  
- 完全に UI 中心の操作でブラケット運用ができる
