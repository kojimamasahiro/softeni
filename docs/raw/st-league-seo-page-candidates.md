# STリーグ SEOページ候補（既存データのみで生成可能）

作成日: 2026-06-21 / 種別: 検討メモ（docs/raw）

## 前提

- 新規データ取得は行わない。`data/st-league/`（editions.json / {2023,2024,2025}/league・participants・matches）と、横断参照可能な `data/players/`・`data/teams/` のみで生成できる候補に限定。
- 既存ページ（重複回避の基準）:
  - `/st-league`（ハブ）、`/st-league/about`（ルール解説）、`/st-league/champions`（歴代王者・記録・昇降格）
  - `/st-league/[year]/matches`（順位表＋対戦結果）、`/st-league/[year]/teams`（出場チーム・選手）、`/st-league/[year]/analysis`（年度別 選手成績）
  - サイト全体: `/teams/[teamId]/...`、`/players/[id]`
- 「推定検索需要」はキーワードツール未使用の相対推定。ソフトテニスは母数が小さい代わりに速報系以外は競合が手薄＝常緑層を狙える、という前提で High/Med/Low を付与。

## 各候補の詳細

### 1. 年度ハブ `/st-league/[year]`（最優先・低工数）
- **なぜ最優先**: 現状 `/st-league/2025` の index が存在せず（matches/teams/analysis サブページのみ）、「STリーグ 2025」「第3回STリーグ」系クエリの受け皿が無い。構造的な穴を埋めつつ、年度内3ページへリンク集約してクロール/リンク評価を改善できる。
- **使うデータ**: `league.json`（会場・期間・format・divisions・results）＋ editions.json（その回のchampions・note）。集計ロジックほぼ不要。
- **内容**: 大会概要（会場・日程・形式）／男女優勝チーム（championsへ）／Ⅰ・Ⅱ部の順位サマリー抜粋＋matches/teams/analysisへの導線。
- **工数: 低**。`[year]/index.tsx` を1枚追加。`getStLeagueYears()` で静的生成（sitemapは自動列挙）。

### 2. チーム別 通年成績 `/st-league/teams/[teamId]`（高価値・中工数）
- **検索意図**: 企業チームの指名検索。STリーグ文脈での「通算成績・優勝回数・所属選手の変遷」は既存の汎用 `/teams/[teamId]`（大会別成績中心）と意図が異なり、重複しにくい。
- **使うデータ**: 3年分 matches（W-L集計）＋ participants（在籍選手の年次変遷）＋ editions/results（優勝・準優勝・昇降格履歴）。すべて既存。
- **内容**: STリーグ通算成績（年度別 勝敗・順位・タイトル）／所属ディビジョン変遷／歴代メンバー→`/players/[id]`。
- **工数: 中**。新規動的ルート＋チーム横断集計。Ⅰ部＋Ⅱ部の対戦データ保有チームに限定（Ⅲ部・女子Ⅱ部は未入力のため除外）すれば薄いページを避けられる。

### 3. 個人成績 歴代ランキング `/st-league/records`（中工数）
- **検索意図**: analysis が年度別のため、通年の勝率・最多勝ランキングは未充足。常緑コンテンツになり得る。
- **使うデータ**: 3年分 matches の D1/S/D2 勝敗を playerId で集計（`buildPlayerMap` 利用）。種目別（ダブルス/シングルス）勝率、通算勝数。
- **内容**: 通算勝率ランキング／最多勝／シングルス勝率トップ など。championsの「記録ハイライト」と相互補完。
- **工数: 中**。横断集計ロジックを新設（既存 `computeRanking` を参考に流用可）。

### 4. 出場選手 索引 `/st-league/players`（内部リンク強化）
- **検索意図**: 「STリーグ メンバー」＋ 選手名ロングテール。主目的は `/players/[id]` への内部リンクハブ化。
- **使うデータ**: 3年分 participants の選手を名寄せ（`normalizeJa` 既存）して五十音/チーム別に列挙。
- **工数: 低〜中**。一覧描画のみ。プロフィールが存在する選手のみリンク。

### 5. 歴代開催地 `/st-league/venues`（任意・低優先）
- editions.json の venue を並べるだけ。単独ページにするほどの需要は薄いため、**championsまたはaboutへのセクション統合**を推奨。

## 推奨ロードマップ

1. **まず #1（年度ハブ）** — 低工数で構造的な穴を埋め、内部リンクの中心になる。
2. 次に **#2（チーム別通年）** と **#4（選手索引）** — 指名・ロングテールを面で取りに行き、既存の `/teams`・`/players` への導線も太くする。
3. **#3（歴代個人ランキング）** — championsと並ぶ常緑の記録系として横断集計を追加。
4. #5 は単独ページ化せず既存ページへ統合。

## 注意 / 制約

- 女子Ⅱ部・全Ⅲ部・2026年は対戦データ未入力（`hasMatchData:false` / サンプル）。チーム・選手・ランキング系は **対戦データ保有スコープに限定** し、薄いページ量産を避ける。
- 純静的ページ（`getStaticProps` を持たない構成）を新設する場合は `next-sitemap.config.js` の `additionalPaths` に追記が必要（動的/SSGは自動列挙）。
- 公開前に docs/wiki/st-league.md の「ページ」節を更新（AGENTSの同期ルール）。
