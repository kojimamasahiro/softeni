# ナビ／レイアウト再設計ドラフト（2ペイン・回遊強化）

- 状態: Draft（討議メモ）
- 日付: 2026-06-22
- 目的: 大会 ↔ 選手 ↔ 年度 のように回遊が深まる導線へ刷新する

> このファイルは docs/raw（未整理の討議メモ）です。確定後に docs/wiki/public-pages.md
> へ反映し、レイアウト構造の決定は ADR を追加します。

---

## 1. 背景・課題

現状ナビは `src/components/Header.tsx` の横並び1本のみ。

- メインサイト: 大会一覧 / 選手一覧 / ニュース / ベータ機能（＋DEV）
- スコアサイト（`isScoreSiteMode()`）: 試合一覧 / 成長分析

課題:

- 横並びのためモバイルで窮屈。折りたたみ（ハンバーガー）が未実装。
- コンテンツは多階層（tournaments / players / teams / highschool / st-league /
  news / growth）に育っているが、グローバルナビが入口の一部しか露出していない。
- エンティティ間（大会↔選手↔年度↔チーム）を行き来する「回遊」導線が
  パンくず＋本文リンク頼みで、面として弱い。

方針決定（2026-06-22 ヒアリング）:

- **B案: 左サイドバー型2ペイン** を採用。回遊が深まることを最優先。
- PC は常設サイドバー、モバイルはドロワー（ハンバーガー）でサイドバーを開閉。

---

## 2. 情報設計（IA）

### グローバル区分（サイドバー第1階層）

実ルート（`src/pages/**`）に基づく入口:

- 大会 `/tournaments`（major / local / 世代・年度・カテゴリ別の深い階層）
- 選手 `/players`
- チーム `/teams`
- 高校 `/highschool`（男女 → 都道府県 → 学校、全国大会歴代記録）
- STリーグ `/st-league`（年度 → teams / matches / analysis / champions）
- ニュース `/news`
- 成長記録 `/growth`（公開ショーケース）
- ベータ機能 `/beta`
- （DEV のみ）ポイント記録 `/beta/matches`

スコアサイトモードは別IA（試合一覧 `getPublicMatchesListPath()` /
成長分析 `getPublicMatchesGrowthPath()`）。**サイドバーは出さず上部バーのみ**
（Q2 決定）。モード分岐は現行どおり `isScoreSiteMode()` を踏襲する。

### サイドバー第2階層（コンテキスト連動）

回遊強化の肝。現在地のセクションに応じてサイドバー下部へ「関連の入口」を出す。

- 大会配下: 主要大会（major）/ 地方大会（local）/ 年度ジャンプ
- STリーグ配下: 年度切替（[year]）/ teams・matches・analysis・champions タブ相当
- 高校配下: 男子/女子セグメント / 都道府県 / 歴代記録
- 選手・チーム配下: 直近大会・年度への横移動

> 第2階層は「サイドバー内に出す」か「本文上部のサブナビに出す」かが論点
> （Open Questions Q3）。

---

## 3. 回遊導線（クロスリンク）の強化

レイアウトと別に、エンティティ相互リンクを面で増やす。

- 大会ページ → 出場選手 `/players/[id]` / 学校 `/highschool/...` / 同大会の他年度
- 選手ページ → 出場大会 / 所属チーム / 成長記録
- 年度（STリーグ [year]・大会 [year]）→ 前年・翌年への前後ナビ
- 既存 `Breadcrumb.tsx`（BreadcrumbList 構造化データ付き）は維持し、
  パンくずの各層を「兄弟への切替」可能なセグメントリンクに拡張する余地あり。
- 公開ページの文言規約（public-pages.md）に従い、内部ファイル名・データ構造名は
  出さず機能として自然に伝える。

---

## 4. レスポンシブ挙動

- PC（lg 以上）: 左サイドバー（幅 ~240px）＋右コンテンツの2ペイン。
  サイドバーは折りたたみ可能（ピン留め、状態保持）。折りたたみ時も
  コンテンツは現状最大幅（max-w-6xl）まで取れる。スコアモードは上部バーのみ。
- タブレット/モバイル: 上部バーにハンバーガー。タップで左からドロワー、
  オーバーレイ表示。`Esc`／オーバーレイクリック／リンク選択で閉じる。
- アクセシビリティ: `aria-expanded` / `aria-controls`、フォーカストラップ、
  `aria-current="page"` で現在地明示。
- ダークモード: 既存の `dark:` クラス方針を踏襲。

---

## 5. 既存コンポーネントへの影響

- `src/components/Header.tsx`: 役割を「上部バー（ロゴ＋ハンバーガー＋最小リンク）」
  に縮小、もしくはサイドバーへ統合。スコアサイト分岐は保持。
- `src/components/PageLayout.tsx`: 現在は中央寄せ1カラム（max-w-*）。
  2ペイン化に伴い「サイドバー＋コンテンツ」のシェル（例: `AppShell`）を新設し、
  PageLayout はコンテンツ側の最大幅制御に専念させる。
- `src/pages/_app.tsx`: `<main>` ラップ箇所。AppShell の挿入位置を確認。
- `src/components/Breadcrumb.tsx`: 維持（構造化データのため）。
- `lib/siteConfig.ts`: モード判定・パス生成ヘルパーをサイドバー構成へ流用。

新規想定:

- `AppShell`（サイドバー＋ドロワー＋上部バーの骨格）
- `SideNav`（第1階層リンク、`aria-current`）
- ナビ定義の単一ソース化（配列定義 → メイン/スコア両モードで分岐）

---

## 6. 段階的実装計画（案）

1. ナビ項目を配列定義として切り出し（メイン/スコア、DEV分岐を内包）。
2. `AppShell` 追加。PC=2ペイン、モバイル=ドロワー。`_app.tsx` で全公開ページに適用。
3. `Header` を上部バーへ縮小、サイドバーへリンク移譲。
4. コンテキスト連動の第2階層（まずは大会・STリーグから）。
5. クロスリンク強化（年度前後ナビ、エンティティ相互リンク）を順次。
6. wiki（public-pages.md）反映＋レイアウト決定の ADR 追加。

## 7. 決定事項（2026-06-22 ヒアリング）

- Q1 → **折りたたみ可能（ピン留め）**。PC サイドバーは展開/折りたたみを
  切替でき、状態を保持する（localStorage 等で記憶）。折りたたみ時は
  アイコンのみのレールか完全非表示かは実装時に詰める。
- Q2 → **スコアサイトモードは上部バーのみ**。サイドバーは出さず、現行の
  上部バー（試合一覧 / 成長分析）を維持する。`isScoreSiteMode()` で
  AppShell を上部バーのみ構成に分岐。
- Q3 → **本文上部サブナビ**。コンテキスト第2階層はサイドバーではなく
  コンテンツ領域の上部に置く。
- Q4 → **コンテンツ最大幅は現状の最大に合わせる**。現状の使用幅は
  4xl（12ページ）/ 5xl（1）/ 6xl（1）、既定 3xl。2ペイン化後も
  コンテンツが現状最大（max-w-6xl）まで取れるシェル幅とし、サイドバーが
  コンテンツを狭めないようにする。各ページの maxWidth 指定は維持。
- Q5 → **年度前後ナビは全エンティティ共通の汎用コンポにする**（可能なら）。
  「前年/翌年」リンクを受け取る汎用 `YearPagerNav`（仮）を新設し、
  STリーグ [year]・大会 [year] 等で共用。前後年度の存在判定は各呼び出し側
  からデータを渡す形にし、コンポは表示と前後リンク生成に専念する。
- Q6 → **既存 SEO 内部導線を再利用・尊重し、重複を作らない**。
  - 既存の意図的導線（都道府県一覧→都道府県→学校、男女セグメント型切替、
    都道府県ページの直近3年表示など）を新ナビで作り直さない。
  - 高校の男女切替は既存 `HighschoolGenderToggle` / セグメント型リンクを
    本文サブナビにそのまま流用する。
  - サイドバーは「セクション入口」に留め、末端ページ（学校ページ等）への
    リンクは本文導線に任せ、サイドバー側で重複させない。
  - 年度前後ナビは、既存の絞り込み意図（例: 直近3年のみ表示）を上書き
    しないよう、適用範囲を限定する。

## 8. 残課題（実装時に詰める）

- 折りたたみ時の表現（アイコンレール vs 完全非表示）と状態保持方法。
- サブナビの共通コンポ化範囲（大会・STリーグ・高校で形が異なるため）。
- `YearPagerNav` の前後存在判定をどのデータ源から渡すか（各セクション差異）。
