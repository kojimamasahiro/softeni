# Softeni Pick UI/情報設計 改善プロジェクト

> **新しいセッション・新しいAIエージェントへ:**
> 必ずこのファイルを最初に読むこと。次に `project-status.md` を読み、
> 現在の状態を把握してから作業を開始する。この2ファイルを読まずに作業してはならない。

## 1. プロジェクト概要

Softeni Pick(ソフトテニスのデータサイト)は機能追加を優先してきた結果、
ページごとに情報構造・UIの一貫性がない。
本プロジェクトは、サイト構造・情報設計・ページ構造・デザイン・コンポーネントを体系的に整理し、
**今後の拡張に耐える設計基盤と運用ルールを確立する**ことを目的とする。

- オーナー(承認者): masahiro
- 進め方: 複数日・複数セッションに分けて実施。担当AIは途中で変わる前提
- 詳細な計画: `project-plan.md`

## 3. 読む順番

1. `PROJECT.md`(本ファイル)
2. `project-status.md` — 現在フェーズ・次回開始位置を確認
3. `rules.md` — 作業ルールを確認
4. `project-plan.md` の**現在フェーズの節のみ**
5. project-status.md の「次回開始位置」に記載された入力ドキュメント(承認済み成果物)

過去フェーズの成果物を全部読む必要はない。現在フェーズの「入力」に指定されたものだけ読む。

## 4. 現在の進捗確認方法

- 進捗は **project-status.md のみ**で管理する。他のファイルに進捗を書かない・探さない
- ドキュメント冒頭のステータス(`draft` / `in-review` / `approved`)で成果物の状態を判別する
- 「承認済み」とは、オーナーが明示的に承認を発言し、decisions.md に記録されたもののみ

## 5. 作業ルール(要約)

詳細は `rules.md`。最低限:

1. フェーズを飛ばさない・並行しない
2. レビュー・承認前に次フェーズへ進まない
3. 次フェーズの入力に使えるのは approved の成果物のみ
4. 決定は decisions.md へ、未決・保留は project-status.md へ即時記録
5. セッション終了時は必ず project-status.md を更新(次回開始位置を含む)
6. 承認判断はオーナーのみ。AIが自己承認しない

各フェーズの目的・入力・レビュー項目・完了条件は `project-plan.md` を参照。

## 7. 次回開始方法

1. 上記「読む順番」の 1〜5 を実行する
2. `project-status.md` の「次回開始位置」に書かれた作業から着手する
3. 着手前に、オーナーへ「現在 Phase N の◯◯から再開します」と一言宣言する(認識ずれの早期検出のため)
4. 作業単位が終わるたびに project-status.md を更新する
5. セッションを終える前に、フェーズ完了時は `templates/phase-report.md` で報告し、
   未完了の場合も「次回開始位置」を必ず具体的に書き残す
