## 1. 移行の原則

1. **移行単位は独立リリース可能**であること。各単位はそれ単体でデプロイでき、途中で止まってもサイトが壊れない
2. **URL 変更を伴うものは 301 + sitemap 更新 + llms.txt 更新をセット**で行う
3. **新旧混在ルール**: 移行中は「ページ単位で新体系へ切替」を基本とし、1ページ内での新旧トークン混在は許可(コンポーネント単位で漸進)。ただしナビ・シェルは一括切替(M2)
4. 各単位の完了時に docs/ui/ の該当ドキュメントへ「実装済み」を記録する

## 3. 順序の根拠

- M1 は無リスクで即実施可能。M2 は入口整備で効果(回遊)が早い
- M3 は SEO 影響が最大のため単独リリースとし、監視期間を確保
- M4→M5→M6 は「構造 → 見た目の基礎 → 部品」の順で、後戻りを防ぐ
- 各単位は 1〜2 セッションで完了する粒度を想定

## 4. 進行中のサイト変更の扱い

- rules.md R15 のとおり: 移行中の機能追加は未決事項へ登録し、現行単位の完了時に影響を反映する
- 特に M3 実施中は記録試合まわりの機能追加を凍結することを推奨
