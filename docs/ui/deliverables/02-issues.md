- 本書の役割: インベントリ(事実)から不整合を**課題として**抽出・分類・優先度付けし、以降のフェーズで解くべき問題を確定する

## 1. 分析方法

- 分類軸: 構造(IA・URL)/ ナビゲーション / 表記・命名 / UIパターン の4分類(project-plan.md Phase 2 準拠)
- 優先度 = 影響度 × 修正コスト
  - 影響度: 高=回遊・拡張性・設計判断に恒常的に影響 / 中=特定領域の一貫性に影響 / 低=限定的・開発時のみ
  - 修正コスト: 高=URL変更やページ再設計を伴う / 中=複数ページの改修 / 低=局所的な修正
  - 優先度: 高=本プロジェクトの中核として解く / 中=該当フェーズで解く / 低=移行計画(Phase 8)で扱うか記録に留める
- 各課題に「解くフェーズ」を割り当てる。Phase 2 では**解決策は設計しない**(スコープはフェーズ定義に従う)
