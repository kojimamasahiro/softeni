## 2. 設計の要点(サマリ)

- **構造変更は最小限・5点のみ**(C-1〜C-5): 記録試合の `/matches` 正規化(/beta 脱却)、/teams 一覧新設、大会入口サブナビ、/highschool 入口化、canonical 文書化。それ以外の既存 URL はすべて温存
- **全公開ページを7タイプに割当**(例外ゼロ)。PageLayout+パンくずを全ページ必須化
- **トークンは Tailwind 標準スケールへの別名方式**(移行容易性優先)。ダークモードは「当面非対応」を提案
- **コンポーネントは base/nav/data/feature の4分類**。PlayerLink/TeamLink の出し分けルールを文書で固定
- **移行は M1(清掃)→ M2(入口)→ M3(URL 正規化・単独リリース)→ M4(構造)→ M5(トークン)→ M6(部品)** の6単位
