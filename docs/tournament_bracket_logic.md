# TournamentBracket ロジック概要

`src/components/Tournament/TournamentBracket.tsx` のロジックに関するドキュメントです。

## 概要
このコンポーネントは、トーナメントの試合データ (`matches`) を受け取り、動的にトーナメント表（ブラケット）を構築して表示します。
固定のHTML構造ではなく、試合結果のつながり（`nextMatchId` や `winnerEntryNo`）を解析して、決勝から逆算してツリー構造を作成し、それをグリッド状に展開して描画しています。

## 1. 使用データ
親コンポーネントから `TournamentBracketProps` として `detailData` を受け取ります。

```typescript
interface TournamentBracketProps {
  detailData: {
    participants: Participant[]; // 選手・チーム情報 (id, name, etc.)
    entries: Entry[];           // エントリー情報 (entryNo, playerIds)
    matches: TournamentMatch[]; // 試合結果情報
  }
}
```

### 主なデータフィールド
- **matches**:
  - `matchId`: 試合の一意識別子
  - `stage`: 'knockout' であるものを対象とする
  - `nextMatchId`: 勝者が進む次の試合ID
  - `winnerEntryNo`: 勝者のエントリー番号
  - `entries`: この試合の対戦ペア（エントリー番号の配列）
  - `round`: ラウンド名（'R1', 'QF', 'SF', 'Final' など）

## 2. ロジック詳細 (`buildBracket`)

ブラケットの構築は `buildBracket` 関数で行われます。

### 手順

1.  **フィルタリング**:
    - `stage === 'knockout'` の試合のみを抽出します。

2.  **ルート（決勝）の特定**:
    - `nextMatchId` を持たない試合を探します。
    - 複数ある場合は、`round === '決勝'` のものを優先します。
    - 見つからない場合は最初の試合を仮定します（エラー回避）。

3.  **ツリートラバーサル（逆探索）**:
    - 決勝戦を `Distance 0` (ルート) とし、再帰的に前の試合 (`prevMatch`) を探索します。
    - **探索ロジック**:
        - 現在の試合の `entries[0]` (上側スロット) と `entries[1]` (下側スロット) について、それぞれ「勝者がそのエントリー番号」かつ「`nextMatchId` が現在の試合ID」である試合を探します。
        - 見つかった場合、その試合を `Distance + 1` として登録します。
    - これにより、`Distance` (決勝からの距離) と `Index` (そのラウンド内での位置) のグリッド (`treeGrid`) が構築されます。

4.  **グリッドの展開 (`entriesByRound`)**:
    - 最も深い `Distance` (1回戦相当) から 0 (決勝) まで順に処理します。
    - 各ラウンドのエントリー配列を作成します。
        - 配列サイズは `2^Distance * 2` です（試合数 × 2）。
    - **シード（Bye）の処理**:
        - 試合が存在しないスロットでも、親ラウンド（`Distance - 1`）にエントリーが存在する場合、それを「シード」として伝播させます。

5.  **優勝者の列**:
    - 決勝戦に `winnerEntryNo` がある場合、最後に「優勝」カラムを追加します。

## 3. 表示ロジック

`TournamentBracket` コンポーネント内で描画を行います。

- **表示ラウンドの制御**:
    - ユーザーがプルダウンで「表示開始ラウンド」を選択可能です（準決勝から表示、など）。
    - `displayedRounds` でフィルタリングされます。

- **レイアウト**:
    - Flexbox (`flex-row`) でラウンドを横に並べます。
    - 各ラウンド内は `flex-col` でエントリーを縦に並べます。
    - **間隔の計算**: `Math.pow(2, roundIndex)` を使用して、ラウンドが進むごとにエントリー間の縦幅（gap）を広げます。これにより、トーナメントのツリー形状が維持されます。

- **エントリー表示**:
    - **最初のラウンド (`showBoxes`)**:
        - エントリー番号、選手名、チーム名を表示するボックスを描画します。
        - 勝者は青色、敗者はグレーで表示されます。
    - **以降のラウンド**:
        - 基本的にボックスは表示せず、接続線のみを表示します（またはコード上の設定による）。
        - *（現在の実装では、最初の表示列のみ詳細ボックスを表示し、以降は線でつながりを示すスタイルになっています）*

- **接続線 (SVG)**:
    - 各エントリーから次のラウンドへの線を SVG で描画します。
    - 勝者のパス（勝ち上がり線）は **青色**、それ以外は **グレー** で描画されます。
    - 試合スコアがある場合、線の上にスコアを表示します。

## まとめ
データ構造としての「試合のリンク（`nextMatchId`）」を正として、表示用のツリー構造を毎回再計算しているのが特徴です。これにより、データが正しい限り、どんな形状のトーナメントでも（シードがあっても）正しく表示されるようになっています。
