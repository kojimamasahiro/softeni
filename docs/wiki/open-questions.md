## 成長分析の公開境界 / 同意

方針（2026-06、ADR-004 Draft）: 成長分析の公開を match の `visibility` とは独立した
ターゲット（個人/ペア）単位の段階公開（L0 none / L1 link / L2 public / L3 ranked）にする。
推奨デフォルトは「既存=L1、新規=グループポリシー（未設定 L1）、L3 は常に opt-in、
個人の撤回が最優先、未成年は保護者同意がなければ L2 上限」。詳細は
[ADR-004](../adr/ADR-004-growth-analysis-visibility-consent.md)。

残る Open Question:

### Q1. 同意主体はグループか本人か
- やりたいこと: 全選手から個別同意を取るのは無理なので、入力者である部活（グループ）が
  「うちの記録は公開可」と一度設定すれば公開できるようにしたい（グループ一次同意）。
- 問題: 部活（顧問・代表）の「公開可」は、本人が「自分の弱点を実名で世間に出される」ことへの
  同意とは限らず、顧問と本人の利害がずれる。本人同意なしに名指し評価データを出し続けてよいか曖昧。
- 決めること: どのレベルまでグループ同意だけで出し、どこから本人同意を必須にするか
  （例: L1 まではグループ同意で可、L2 以上は本人同意必須）。

### Q2. 未成年・保護者同意の取得と確認
- やりたいこと: コンテンツの中心は中高生なので、学生も L2/L3 で公開・ランキング掲載したい。
- 問題: 未成年は本人同意だけでは足りず保護者同意が要る場面があるが、部活が一括入力する運用で
  多数の保護者同意を集める手段がない。`guardian_consent` を作っても誰がどう立て・確認するのか不明。
- 決めること: 保護者同意を集め・確認する手段を作るのか。作れないなら
  「未成年は L3 を匿名のみに制限」と割り切って回避するのか。

### Q3. ランキング/集約での実名 vs 匿名の既定
- やりたいこと: 「最も伸びた選手」ランキングや伸び率の集約記事を作りたい（新規流入の目玉）。
- 問題: 実名のほうがコンテンツとして強く SEO も効くが、同意ハードルが上がり未成年問題に直撃する。
  匿名（地域・カテゴリ・イニシャル）なら同意は緩むが訴求は弱い。
- 決めること: 集約コンテンツの既定を実名にするか匿名にするか（推奨は既定匿名＋実名 opt-in）。
  それで記事として成立する魅力が残るかを判断する。

### Q4. 既存ターゲットを L1 に倒した後の引き上げ導線
- やりたいこと: 既存データは安全側で L1 に倒すが、そのままでは公開コンテンツが空になるので、
  同意を取って L2 以上へ引き上げたい。
- 問題: 既に記録済みの過去の選手に「公開してよいか」を聞く連絡手段（メアド等）を持っていない。
- 決めること: 誰に・どの画面で・どうやって引き上げ同意を取るか
  （部活経由で声かけ／サイト上に「自分のページを公開申請」導線、など）。

### Q5. 撤回反映の SLA
- やりたいこと: 「やっぱり消して」に応える撤回（オプトアウト）を用意したい。
- 問題: サイトは静的生成のため、撤回を DB に書いても次のビルド＆デプロイまで実ページは消えない。
  炎上・いじめ・個人情報懸念で「今すぐ消したい」場合、ビルド周期では遅すぎる。
- 決めること: 撤回を何時間以内に反映すると約束するか。緊急削除用に該当 JSON 即時削除／CDN パージの
  別経路を持つか。

### Q6. score mode（UGC）転換時の同意モデル統合
- やりたいこと: ADR-003 Phase 2 で score 側を UGC（ログインユーザーが自分の試合を記録）の本拠地にする計画があり、
  そこでは選手＝ログイン本人。
- 問題: 本方針の `growth_consent` は `subject_key`＝正規化氏名のグループ起点。UGC では同意主体が
  `owner_user_id`（アカウント本人）になり、氏名ベース同意とアカウント同意の 2 モデルが併存・衝突する。
- 決めること: 認証ユーザー所有モデル導入時に、`growth_consent` をどうアカウントへ紐づけ直すか（移行方針）。
