# 高校SEO M4検証ランブック（GSC事後検証）

2026-07-17〜18 に実施した高校カテゴリのSEO施策一式が「実際に検索で効いたか」を
Google Search Console（GSC）で検証するための手順書。**このページ単体で、当時の文脈を
覚えていなくても実行できる**ことを目的に書いている。背景の詳細は
[raw/2026-07-17-idea-highschool-strong-school-ranking.md](../raw/2026-07-17-idea-highschool-strong-school-ranking.md)
と [raw/2026-07-17-highschool-head-query-seo.md](../raw/2026-07-17-highschool-head-query-seo.md)。

## 背景（何をやったか）

出発点: Google Trends で「ソフトテニス 高校」が最頻の組み合わせクエリなのに、
softeni-pick は表示ゼロだった（2026-07-17 時点。SERP は各県高体連公式が独占）。
正面から2語ヘッドを取るのは困難なため、「強豪/ランキング/歴代/出身/県別」系の
3語クエリを面で取り、積み上げでヘッドにも表示を発生させる戦略を採った。

実施済み施策（すべて2026-07-17〜18、直後にデプロイ・インデックス登録リクエスト済み）:

1. タイトル修正: `/highschool/[gender]` と都道府県ページに「高校◯子ソフトテニス」の隣接一致、都道府県ページに「強豪校」
2. `/highschool/rankings/` 新設: 学校別トップ100＋都道府県別47ポイントランキング（独自配点。非公式の注記あり）
3. 全日本高校選抜 2020〜2025年度を収録し、歴代記録ページ `/highschool/tournaments/senbatsu/` を追加（汎用ハブ `/tournaments/highschool/highschool-senbatsu` は noindex, follow 化）
4. 都道府県ページに「県内強豪校」上位5校セクション＋県別FAQ
5. 学校ページに「主な卒業生」セクション＋FAQ（約64学校×性別に表示）
6. ランキング⇔歴代記録の相互リンク
7. インターハイ2026 preview・結果即日反映（M1。ユーザー側で実施）

## いつやるか

- **初回: 2026年8月中旬**（デプロイ+約4週。インターハイ検索スパイク期を含む）
- 2回目: 2026年9月末（平常期にどれだけ残るか）
- 選抜前チェック: 2027年2月（3月の選抜検索期に向けた仕込み確認）

GSC のデータは2〜3日遅れで反映される。また少数クエリはプライバシーで非表示になるため、
「表示されないクエリ＝ゼロ」とは限らない点に注意。

## 手順

### 0. 前提

- GSC プロパティ: `softeni-pick.com`
- 比較期間: 「過去28日」vs「前の期間」。初回はデプロイ前（〜2026-07-17）との比較になるようにする

### 1. インデックス確認（URL検査）

以下を URL 検査にかけ、「インデックス登録済み」かを記録する。未登録なら登録リクエスト。

- `https://softeni-pick.com/highschool/`
- `https://softeni-pick.com/highschool/rankings/`
- `https://softeni-pick.com/highschool/tournaments/senbatsu/`
- 都道府県ページのサンプル2〜3件（例: `/highschool/boys/nara/`、`/highschool/girls/miyagi/`）
- 学校ページのサンプル2〜3件（例: `/highschool/boys/nara/takadashou/`、卒業生セクションがある校）
- **逆チェック**: `/tournaments/highschool/highschool-senbatsu`（汎用ハブ）が「noindex により除外」になっていること

### 2. クエリ群の計測（検索パフォーマンス）

「検索パフォーマンス」→ 新規タブでクエリフィルタを掛け、表示回数・クリック・平均掲載順位を記録する。

計測するクエリ群（正規表現フィルタが使える場合は「カスタム（正規表現）」）:

| 狙い | クエリ例（含むフィルタ） | 主な受け皿ページ |
|---|---|---|
| ヘッド（最終目標） | `ソフトテニス 高校` / `高校 ソフトテニス` | /highschool/ 一式 |
| 強豪・ランキング | `強豪`、`ランキング` | /highschool/rankings/ |
| 県別強豪 | `{県名} 高校 ソフトテニス`（主要数県で確認） | 都道府県ページ |
| 強い県 | `強い県`、`都道府県` | /highschool/rankings/ |
| 選抜歴代 | `高校選抜`、`選抜 ソフトテニス` | /highschool/tournaments/senbatsu/ |
| 出身・OB | `出身`、`OB` | 学校ページ |
| 学校名 | `高田商業 ソフトテニス` など数校 | 学校ページ |

あわせて「ページ」フィルタでも確認する: `/highschool/rankings/`、`/highschool/tournaments/`、
都道府県ページ（URL に `/highschool/boys/` `/highschool/girls/` を含む）、学校ページ。

### 3. カニバリ確認（seo.md の宿題の答え合わせ）

同一クエリに複数URLが表示されていないかを見る（クエリを選択→「ページ」タブ）。

- **seo.md #3**: 「ソフトテニス 高校選抜」系で `/highschool/tournaments/senbatsu/` に集約されているか。noindex にした汎用ハブが表示されていないか
- **seo.md #7**: 「◯◯高校 ソフトテニス メンバー/出身」系が学校ページに集約されているか
- **seo.md #10**: 「{県名} 高校 ソフトテニス 強豪」で都道府県ページとランキングページが分散していないか

### 4. 判定基準と次のアクション

| 観測結果 | 判定 | 次のアクション |
|---|---|---|
| ランキングページが「強豪/ランキング」系で表示あり・順位20位以内 | 成功 | 県別展開・公立/私立フィルタの実装判断へ（Backlog参照） |
| 同・表示はあるが50位以下 | コンテンツ一致不足 | 実際に表示されたクエリ文言に合わせて title/h1/FAQ を微調整 |
| 同・表示ゼロ（4週後） | インデックス/評価の問題 | 手順1を再確認→内部リンク増強（トップページからの導線等）を検討 |
| 「ソフトテニス 高校」ヘッドで表示発生 | 戦略が機能 | 表示クエリと受け皿URLを記録。次はCTR（title改善）へ |
| #10 で2URLに分散 | インテント分割失敗 | seo.md #10 を見直し、どちらかへ寄せる（title から「強豪校」を片方外す等） |
| 汎用ハブ（highschool-senbatsu）が表示されている | noindex 未反映 | URL検査で noindex 認識を確認、必要ならサイトマップ・キャッシュ確認 |
| 卒業生系クエリが観測されない | 想定内（超ロングテール） | 静観。学校名クエリでの学校ページ順位が上がっていればOK |

### 5. 記録（必須）

結果は `docs/raw/YYYY-MM-DD-m4-gsc-review.md` を新規作成して記録し、
[seo.md](./seo.md) の #3 / #7 / #10 の「状態」欄を実測に基づいて更新する
（監視→対策済、または見直しへ）。raw ノートのテンプレ:

```markdown
# M4 GSC検証 第N回（YYYY-MM-DD）

## 計測期間: YYYY-MM-DD〜YYYY-MM-DD（比較: 前期間）
## インデックス状況: （手順1の結果）
## クエリ群の結果: （手順2の表をそのまま埋める。表示/クリック/順位）
## カニバリ確認: （手順3の結果。#3/#7/#10それぞれ）
## 判定と次のアクション: （手順4の表に当てはめた結論）
## seo.md 更新箇所: （更新した項目）
```

## 検証後の「次の一手」候補（優先度順・Backlog済み）

1. 県別強豪クエリが取れていれば: ランキングの県別展開・公立/私立属性フィルタ（[highschool.md](./highschool.md) Backlog）
2. 国体/国スポ 少年の部のデータ収録（M3残り。都道府県選抜チーム制のため学校帰属の設計判断が必要）
3. インターハイ過去年度（2015〜2020）の遡り収録（歴代ページの厚み・「インターハイ {年} 優勝」系ロングテール）
4. 被リンク獲得: X での DB 由来の記録トリビア発信（rare-events の高校版。例: 「選抜4連覇は史上◯回目」）
