# SNS 1日目投稿画像（sns-images / day1）

## 概要

2日制の大会で「1日目終了時点」にXへ投稿し、2日目の**集客**につなげるための画像・キャプション自動生成の仕様。
ツール本体は `tools/sns-images/`。依存は Pillow のみ。設計議論は `docs/raw/2026-06-16-hjc-day1-sns-design.md`。

方針: **手元の内部データ（`data/tournaments/**`）から客観的事実として出せるものは全て自動化\*\*する。
外部データ（配信URL・コート割・リアルタイム進行）には依存しない。

## 対象大会の構造（ハイスクールジャパンカップ 2026）

スケジュール（2026）:

- 6/26(金) シングルストーナメント（1回戦〜**準々決勝終了**）
- 6/27(土) シングルス準決勝〜決勝（→表彰式）／表彰式後にダブルス予選リーグ開始
- 6/28(日) ダブルス決勝トーナメント（→表彰式・閉会式）

これにより「1日目」の意味が種目で異なる:

| 種目       | 1日目終了時点                                            | 2日目                     |
| ---------- | -------------------------------------------------------- | ------------------------- |
| シングルス | ベスト4確定（準々決勝まで終了。knockoutのみ）            | 6/27(土) 準決勝〜決勝     |
| ダブルス   | 予選リーグ終了・**本選進出ペア確定**（roundrobin段のみ） | 6/28(日) 決勝トーナメント |

注: シングルスは土曜で完結するため、土曜は「ダブルス1日目投稿」＋「シングルス完結報告」が並ぶ。

## データ仕様（確認済み）

入力は `data/tournaments/details/<大会>/<年>/<カテゴリ>.json`。未消化試合は `winnerEntryNo: null`。

- `matches[]`: `stage`（`knockout` / `roundrobin`）, `round`, `group`, `entries`, `scores`, `winnerEntryNo`, `nextMatchId`, `prevMatchIds`。
- `entries[]`: `entryNo`, `playerIds`（個人=1, ダブルス=2）。
- `participants[]`: `id`（`姓_名_所属_県`）, `lastName`, `team`, `prefecture`。
- `results[]`: `tournament.rank`（最終成績）, `roundrobin.{group,rank}`（リーグ順位）。※入力途中JSONには無い場合があるため、**順位はmatch群から自前計算**する（下記）。

### ダブルス 本選進出ペアの判定（自前計算）

- グループ（`group`）ごとに roundrobin の完了試合を集計。
- 順位付け: **勝数 → ゲーム差(得-失) → 総得ゲーム** の降順。各グループ**1位のみが本選進出**。
- 確認: HJC2025 男女ダブルスとも、この計算による各グループ1位が `results[].roundrobin.rank==1`（=knockout出場24ペア）と完全一致。
- 24グループ→24進出ペア。`results` が無い入力でも match群だけで再現可能。

### 前年実績proxy（番狂わせ／生存の客観指標）

シード情報が無いため、**前年（直近1年）成績**で代替する。

- 「前年ベスト8以上」= 前年同カテゴリの `results[].tournament.rank` が
  `winner` / `runnerup` / `best(bestLevel<=8)` のいずれか（=上位8エントリ）。
- 対象選手= 前年の該当エントリの `playerIds`（**選手id単位**で保持。ダブルスはペア再編に対応するため選手単位）。
- 今年データで、当該選手idを含むエントリが「1日目終了時点で生存／敗退」を判定して提示。
- 基準は**前年のみ**・閾値は**ベスト8以上**（確定事項）。

## 出力方針（バランス：画像は主役カットのみ）

画像は「文字で代替しにくい主役カット」だけにし、サマリ・番狂わせ・結果ダイジェストは
**投稿本文（caption）＋スレッド返信（thread）のテキスト**で出す。テキストは
`<--out>/captions/<カテゴリ>.txt`（本文）と `<カテゴリ>_thread.txt`（返信）に生成する
（例: `--out out/sns` なら `out/sns/captions/`）。

- 氏名は**フルネーム**（姓＋名）。所属は `学校（都道府県）`。
- 画像サイズはX向けに **16:9(1200×675) / 1:1(1200×1200) / 4:5(1200×1500)** の3種に統一。
  中身は縦中央寄せ。主役カットは16:9、密な一覧は4:5。

### A. シングルス1日目（ベスト4確定）

- 画像: ①表紙＋CTA（16:9） ②ベスト4一覧（16:9） ③準決勝 確定カード（16:9）
- 本文: ベスト4決定・勝ち残り県・CTA・ハッシュタグ
- スレッド: 準々決勝結果＋前年ベスト8以上の敗退/生存

### B. ダブルス1日目（予選リーグ終了）

- 画像: ①表紙＋CTA（16:9） ②本選進出ペア一覧（4:5・1〜2枚）
- 本文: 進出ペア数・県別上位・CTA・ハッシュタグ
- スレッド: 前年ベスト8以上の予選敗退/通過・複数進出校
- リーグ星取表・進出サマリ画像は出さない。

### C. シングルス完結報告（土・記録/反応狙い）

- 画像: ①優勝（16:9） ②準優勝・ベスト4（16:9）
- 本文: 優勝者・決勝スコア・ハッシュタグ
- スレッド: ベスト8の都道府県・準決勝スコア

## CTA 方針（内部データ限定）

- 会場（`prefecture`/`location`）は `data/tournaments/information/<大会>.json` の該当年レコードから取得。
- **2日目の日付は種目で異なる**（シングルス=土、ダブルス=日）。情報JSONは大会全体の開始/終了日しか持たないため、
  2日目日付は実行時に CLI 引数（`--next-date`）で渡す（運用時に判明している客観事実。永続的な手動メタファイルは作らない）。
- 配信URL・コート割・開始時刻詳細は**載せない**（手元に無い／外部依存）。
- 情報JSONに該当年レコードが無い場合（例: 2026は未登録）は会場を省略して描画する。

## CLI（実装済み）

会場（CTA）の描画には情報JSON参照のため `--tournament` / `--year` が必要。
個別スクリプトは details.json を位置引数で受ける。

```bash
# ダブルス1日目（本選進出）
python tools/sns-images/day1_doubles.py <details.json> \
    --next-date "6/28(日)9:00〜" --tournament highschool-japan-cup --year 2026 --out out/sns

# シングルス1日目（ベスト4）
python tools/sns-images/day1_singles.py <details.json> \
    --next-date "6/27(土)9:00〜" --tournament highschool-japan-cup --year 2026 --out out/sns

# シングルス完結報告
python tools/sns-images/result_singles.py <details.json> --out out/sns

# 4カテゴリ一括＋キャプション生成
python tools/sns-images/day1_all.py --tournament highschool-japan-cup --year 2026 \
    --singles-next "6/27(土)9:00〜" --doubles-next "6/28(日)9:00〜" --out out/sns
```

各投稿のキャプション（客観テンプレ＋ハッシュタグ）は `captions/<カテゴリ>.txt` に併せて出力する。

## 確定事項（2026-06-16）

- 番狂わせ: 前年のみ・ベスト8以上・選手id単位。
- CTA: 内部データ限定（会場は情報JSON、2日目日付はCLI）。手動メタファイルは作らない。
- 荒天時ダブルス=トーナメント変更の自動判定: 実装しない（手動対応）。
- ダブルスのリーグ星取表描画: 不要（進出ペア一覧重視）。

## 検証方法

完了済み大会（例 2025）の details から後半の `winnerEntryNo` を null 化し「1日目途中」を再現して画像生成・目視確認する。

- シングルス: 準決勝・決勝を null 化 → ベスト4確定状態。
- ダブルス: knockout 全試合を null 化 → 予選リーグ終了状態。
